3. Progressive Disclosure — Three-Tier Token Strategy

The core pattern: load minimum context first, expand only when needed.

graph TD
  subgraph T1["TIER 1 — Catalog (Always Loaded)"]
    T1_WHAT["• Skill name + description only
• Injected into system prompt every turn
• ~50 tokens per skill
• 20 skills = ~1,000 tokens"]
  end

  subgraph T2["TIER 2 — Instructions (On Activation)"]
    T2_WHAT["• Full SKILL.md body
• Loaded via activate_skill() tool call
• Returns as tool_result in message history
• Recommended <5,000 tokens per skill"]
  end

  subgraph T3["TIER 3 — Resources (On Demand)"]
    T3_WHAT["• Reference files: guides, checklists, examples
• Scripts: executed, only output enters context
• Loaded via read_resource() or file-read tools
• Unbounded size, only pay for what you read"]
  end

  T1 -->|"LLM matches user intent
→ calls activate_skill(name)"| T2
  T2 -->|"SKILL.md references a file
→ LLM reads it on demand"| T3

4. Skill Activation — Full Sequence

End-to-end flow: user message → skill activation → resource loading → response.

sequenceDiagram
  participant U as User
  participant A as Agent Runtime
  participant R as Skill Registry
  participant LLM as LLM
  participant FS as Filesystem

  Note over A: Startup
  A->>FS: Scan .agents/skills/*/SKILL.md
  FS-->>A: Found: code-review, deploy, research
  A->>R: Parse & cache all skills

  Note over A: Build System Prompt
  A->>R: Get catalog (names + descriptions)
  R-->>A: [{name, desc}, ...]
  A->>LLM: System prompt with catalog + tool schemas

  Note over U,LLM: User Interaction
  U->>A: "review this PR for security issues"
  A->>LLM: Forward user message

  Note over LLM: Intent Matching
  LLM->>LLM: Catalog says "code-review: Review code for bugs and security"
  LLM->>A: tool_call: activate_skill(name="code-review")

  Note over A: Tier 2 — Load Instructions
  A->>R: Lookup "code-review"
  R-->>A: Skill{body: "## Code Review\n\n...", resources: ["security-rules.md", ...]}
  A->>A: Mark "code-review" as active (dedup tracking)
  A->>LLM: tool_result: {content: full_body, resources: [...]}

  Note over LLM: Follow Skill Instructions
  LLM->>LLM: Reads: "For security reviews → read security-rules.md"
  LLM->>A: tool_call: read_resource(skill="code-review", file="security-rules.md")

  Note over A: Tier 3 — Load Resource
  A->>FS: Read .agents/skills/code-review/security-rules.md
  FS-->>A: File content
  A->>LLM: tool_result: security rules content

  Note over LLM: Execute with Full Context
  LLM->>LLM: Has: skill instructions + security rules
  LLM->>U: Detailed security-focused code review

6. Context Window Management

How skill content survives in long conversations — dedup, compaction protection, lifecycle.

stateDiagram-v2
  [*] --> Cataloged: Startup scan

  Cataloged: 📋 Cataloged
  Cataloged: name + desc in system prompt
  Cataloged: 0 tokens in message history

  Active: ⚡ Active
  Active: Full body in message history
  Active: Protected from compaction
  Active: Tracked in active_skills set

  ResourceLoaded: 📂 Resource Loaded
  ResourceLoaded: Reference file content
  ResourceLoaded: in message history
  ResourceLoaded: May be compacted (not protected)

  Cataloged --> Active: LLM calls activate_skill()
  Active --> Active: LLM calls activate_skill() again\n(DEDUP — skip, already loaded)
  Active --> ResourceLoaded: LLM reads reference file
  ResourceLoaded --> ResourceLoaded: LLM reads another file

  note right of Active
    COMPACTION RULES:
    ─────────────────
    ✅ Compress: old user/assistant messages
    ✅ Compress: resource file content (Tier 3)
    ❌ Never compress: active skill body (Tier 2)
    ❌ Never compress: system prompt catalog (Tier 1)
  end note

7. Data Model (Pydantic / Any Language)

Core types for implementation. Framework-agnostic.

classDiagram
  class Skill {
    +str name
    +str description
    +str body
    +Path location
    +str source: "project" | "user"
    +dict metadata
    +list~str~ allowed_tools
    +list~str~ resources
  }

  class SkillRegistry {
    -dict~str,Skill~ _skills
    -float _last_scan_ts
    -float _cache_ttl
    +discover(workspace: Path) list~Skill~
    +get(name: str) Skill | None
    +catalog() list~SkillSummary~
    +list_resources(name: str) list~str~
    +read_resource(name: str, file: str) str
  }

  class SkillSummary {
    +str name
    +str description
  }

  class ContextManager {
    -set~str~ _active_skills
    +is_active(name: str) bool
    +mark_active(name: str) void
    +should_compact(msg: Message) bool
  }

  class PromptAssembler {
    +render_catalog(skills: list~SkillSummary~) str
    +render_discovery_rules() str
    +build_system_prompt(base: str, catalog: str) str
  }

  class ToolHandler {
    +activate_skill(name: str) ToolResult
    +read_resource(skill: str, file: str) ToolResult
    +run_script(skill: str, script: str, args: dict) ToolResult
  }

  SkillRegistry "1" --> "*" Skill : stores
  SkillRegistry --> SkillSummary : produces
  ToolHandler --> SkillRegistry : queries
  ToolHandler --> ContextManager : updates
  PromptAssembler --> SkillRegistry : reads catalog
  ContextManager --> PromptAssembler : informs compaction
